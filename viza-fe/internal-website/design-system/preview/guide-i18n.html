Rules
Line-height for CJK

Mandarin sets denser than Latin. Bump body line-height from 1.5 → 1.7, headings from 1.3 → 1.4 when the content language is Chinese. Letter-spacing should be 0 — the global -0.011em cascade was tuned for Latin.

Button width tolerance

Chinese buttons can be 40–60% narrower than English. Never fix button widths — always width: fit-content with consistent horizontal padding.

Input font-size ≥ 16px

Prevents iOS Safari auto-zoom. Applies to every input, not just the first one on the page.

Punctuation

zh-CN uses full-width punctuation (，。？！「」) inside prose. In UI labels, prefer en-dashes and middle dots (·) over slashes for readability in both scripts.

Fonts

Latin headings → Switzer. Chinese headings → PingFang SC (macOS/iOS) / Noto Sans SC (Android/web). Body is Geist for Latin, PingFang SC for Chinese. Never mix families mid-word — let the fallback stack handle it.

RTL

Not supported. Arabic/Hebrew locales are out of scope for the current product. Don't build RTL-ready — build LTR correctly.
